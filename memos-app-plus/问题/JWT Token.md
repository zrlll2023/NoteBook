JWT 是什么：服务器登录成功后颁发的一个「加密字符串」，里面包含用户ID、过期时间等信息。之后每次请求带上它，服务器不需要查数据库就能知道你是谁。

为什么叫「无状态」：服务器不需要记录「谁登录了」，Token 本身就携带了信息。

退出登录为什么需要 Redis 黑名单：Token 发出去就没法改，所以用 Redis 单独记录「已登出的 Token」，每次请求来先查一下黑名单。

JWT 的完整核心思路/整个退出登录的流程：
1.用户点退出登录
2.前端发请求，请求里带着（用户主动退出时提交的Token/已登出的Token）
3.后端收到请求，把这个（用户主动退出时提交的 Token/已登出的Token）存进 Redis 黑名单
4.之后坏人拿着这个旧 Token 来请求，后端去 Redis 查一下，发现它在黑名单里，直接（拒绝一切访问）

token是经过 算法信息（这里用的是HMAC256的算法信息） + .withClaim里面的内容 + 你的密钥（也可以是任何内容，统称为密钥）  经过算法(这里用的是HMAC256的算法)计算出来的

算法信息：  
官方名字叫 Header，它其实是一小段 JSON 数据，用来告诉系统两件事：  
1：令牌的类型：固定是 JWT。
2：使用的加密算法：也就是你选择的签名方法。  
框架会自动把这段信息用 Base64 编码，变成 Token 的第一段长字符串。  
```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

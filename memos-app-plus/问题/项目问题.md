#### 为什么 TINYINT(1) 不用 boolean？
技术上用 `boolean` 也能跑，但 `status` 这个字段将来可能不只是 0 和 1——比如以后加一个「2 = 注销」的状态，`boolean` 就装不下了。用 `Integer` 更灵活，这是一个工程习惯。

#### Java 里 interface 和 class 的区别
`interface` 是一份**合同**，只规定「你必须能做什么」，但不写具体怎么做。 `class` 是**具体的实现**，真正把事情做了。
比如：
```
// 合同：规定必须有这个能力
interface 会飞 {
    void fly();
}

// 具体实现：真正写怎么飞
class 鸟 implements 会飞 {
    void fly() {
        System.out.println("扇翅膀飞");
    }
}
```
Mapper 用 `interface` 是因为——具体怎么操作数据库，是 MyBatis-Plus 在背后帮你实现的，你只需要定义「我要有这个能力」就够了。

这其实就是 **Spring IOC** 的体现——你不用自己 `new` 一个 Mapper 对象，Spring 帮你创建并管理它，你只需要声明你需要它。

Mapper 做的事情是：

> 「我已经知道 `User` 长什么样了（entity 定义好了），现在请给我**操作 `users` 表的增删改查能力**。」

MyBatis-Plus 看到 `extends BaseMapper<User>`，就自动把增删改查的具体实现注入进来。

所以分工是：

- `entity` → 定义表结构（每列是什么）
- `mapper` → 声明我需要操作这张表的能力
- `MyBatis-Plus` → 在背后帮你实现具体的 SQL
